OPEN SOURCE • CONTROL LAYER

The Control Layer.
Hard limits for autonomous code.

Agents are probabilistic. Your infrastructure is not. The Control Layer wraps every agent with enforceable constraints—budgets, step limits, tool access, loop detection, rate limits, and circuit breakers. When something goes wrong, Splinter kills it before damage is done.

Budgets & Limits

Hard caps on time, memory, token spend, and step count per agent. Per-tool budgets so one expensive call can't drain your entire allocation. Threshold behavior configurable to warn or stop.

Loop Detection & Retry Strategy

Detect when an agent repeats the same action and break the loop automatically. Retry strategies per tool—basic retries for transient failures, fail-closed for anything destructive.

Tool Access & Custom Rules

Define exactly which tools each agent can use. Write custom rules to gate dangerous operations. Rate limit per agent per tool so no single worker monopolizes shared resources.

Decision Enforcement

Once an agent makes an important decision, Splinter locks it. The agent cannot change its mind mid-execution. No more waffling where plans get rewritten on every iteration.

Snapshots & Rollback

Every action is snapshotted. If something goes wrong, roll back to the last safe checkpoint. No need to restart the entire chain from scratch.

Circuit Breakers

Global safety triggers that halt the entire swarm when critical thresholds are hit—runaway spend, cascading failures, or anomalous behavior. The emergency stop for your agent fleet.

The Coordination Layer.
Agents that know about each other.

Multi-agent systems fail when agents act in isolation. The Coordination Layer gives every agent awareness of the full chain—shared state with clear ownership, validated handoffs, sequential checkpoints, goal tracking, and explicit signals for when steps are done or why an agent is waiting.

Shared State & Ownership

Every piece of shared state has a designated owner. Splinter tracks who wrote what and prevents unauthorized overwrites. Agent B can read plan_output but cannot modify it unless ownership is explicitly transferred.

Handoff & Schema Validation

When one agent hands off to another, Splinter validates the payload against a declared schema. If validation fails, the handoff is rejected and the upstream agent is notified. No more passing garbage context downstream.

Chain Awareness & History

Every agent knows what came before it. Splinter injects the execution history of upstream agents so downstream workers have full context. No more agents operating blind in a pipeline they don't understand.

Goal Awareness & Validation

Each agent has a declared goal. Splinter validates that the agent's output actually satisfies its stated objective before allowing the step to be marked complete. Goals prevent drift.

Action Eligibility & Completion Signals

Only the eligible agent can act at any moment. Steps must explicitly declare when they're done. Idle agents must declare what they're waiting for. No ambiguity, no race conditions, no silent stalls.

Checkpointing & Sequential Pipelines

State is checkpointed at every pipeline step. If a downstream agent fails, resume from the last checkpoint instead of rerunning everything. Sequential order is enforced—no step runs before its predecessor completes.
