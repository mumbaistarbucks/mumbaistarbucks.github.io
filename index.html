OPEN SOURCE • CONTROL LAYER

The Control Layer.
Hard limits for autonomous code.

Agents are probabilistic. Your infrastructure is not. The Control Layer wraps every agent with enforceable constraints—budgets, step limits, tool access, loop detection, rate limits, and circuit breakers. When an agent goes off the rails, Splinter stops it before damage is done.

Budgets & Limits

Set hard caps on time, memory, token spend, and step count per agent. Assign per-tool budgets so a single expensive tool call can't drain your entire allocation. Configure threshold behavior to warn or stop when limits approach.

Loop Detection & Retry Strategy

Detect when an agent is stuck repeating the same action and break the loop automatically. Configure retry strategies per tool—basic retries for transient failures, fail-closed for anything destructive.

Tool Access & Custom Rules

Define exactly which tools each agent can access. Write custom rules to gate dangerous operations. Rate limit tools per agent so no single worker monopolizes shared resources.

Decision Enforcement

Once an agent makes an important decision, Splinter locks it. The agent cannot change its mind mid-execution. This prevents the "waffling agent" problem where plans get rewritten on every iteration.

Snapshots & Rollback

Every action is snapshotted. If something goes wrong, roll back to the last safe checkpoint instantly. No need to restart the entire chain from scratch.

Circuit Breakers

Global safety triggers that halt the entire swarm when critical thresholds are hit—runaway spend, cascading failures, or anomalous behavior. The emergency stop for your agent fleet.

splinter.control.ts
import { Splinter } from '@splinter/core'

const agent = Splinter.agent({
name: 'Executor',

// Hard limits
budget: { tokens: 5000, time: '30s' },
maxSteps: 10,
budgetThreshold: { at: 0.8, action: 'warn' },

// Tool access & rate limits
tools: ['fs.read', 'fs.write'],
rateLimit: { tool: 'fs.write', max: 3, per: '1m' },

// Safety
loopDetection: true,
retryStrategy: 'fail-closed',
decisionEnforcement: true,
circuitBreaker: {
trigger: 'spend > $5',
action: 'halt-all'
}
})

The Coordination Layer.
Agents that know about each other.

Multi-agent systems fail when agents act in isolation. The Coordination Layer gives every agent awareness of the chain—shared state with clear ownership, validated handoffs, sequential checkpoints, goal tracking, and explicit signals for when steps are done or why an agent is waiting.

Shared State & Ownership

Every piece of shared state has a designated owner. Splinter tracks who wrote what and prevents unauthorized overwrites. When Agent A owns plan_output, Agent B can read it but cannot modify it unless ownership is explicitly transferred.

Handoff & Schema Validation

When one agent hands off to another, Splinter validates the handoff payload against a declared schema. No more passing malformed or incomplete context between agents. If validation fails, the handoff is rejected and the upstream agent is notified.

Chain Awareness & History Visibility

Every agent knows what came before it. Splinter injects the execution history of upstream agents so downstream workers have full context. No more agents operating blind in a pipeline they don't understand.

Goal Awareness & Validation

Each agent has a declared goal. Splinter validates that the agent's output actually satisfies its stated objective before allowing it to mark the step as complete. Goals prevent agents from going off-task.

Action Eligibility & Completion Signals

Only the eligible agent can act at any moment. Steps must explicitly declare when they're done. When an agent is idle, it must declare what it's waiting for. No ambiguity, no race conditions, no silent stalls.

Checkpointing & Sequential Pipelines

State is checkpointed at every pipeline step. If a downstream agent fails, resume from the last checkpoint instead of rerunning the entire chain. Sequential order is enforced—no step can run before its predecessor completes.
