Open Source • Control Layer

The Control Layer.
Hard limits for autonomous code.

Every agent gets enforceable constraints. Budgets, step limits, tool access, loop detection, rate limits, and circuit breakers. When something goes wrong, Splinter kills it before damage is done.

Budgets & Limits

Hard caps on time, memory, tokens, and steps per agent. Per-tool budgets. Configurable thresholds that warn or stop.

Loop Detection

Detects repeated actions and breaks the loop. Retry strategies per tool — basic for transient failures, fail-closed for destructive ones.

Tool Access & Rules

Define which tools each agent can use. Custom rules to gate dangerous operations. Rate limits per agent per tool.

Decision Enforcement

Once an agent makes an important decision, Splinter locks it. No more waffling where plans get rewritten every iteration.

Snapshots & Rollback

Every action is snapshotted. Roll back to the last safe checkpoint instantly. No need to restart the entire chain.

Circuit Breakers

Global safety triggers that halt the entire swarm — runaway spend, cascading failures, or anomalous behavior.

Open Source • Coordination Layer

The Coordination Layer.
Agents that know about each other.

Shared state with clear ownership, validated handoffs, sequential checkpoints, goal tracking, and explicit signals for when steps are done or why an agent is waiting.

Shared State & Ownership

Every piece of state has a designated owner. Splinter tracks who wrote what and prevents unauthorized overwrites.

Handoff & Schema Validation

Handoff payloads are validated against a declared schema. If validation fails, the handoff is rejected and the upstream agent is notified.

Chain Awareness

Every agent knows what came before it. Execution history from upstream agents is injected so downstream workers have full context.

Goal Validation

Each agent has a declared goal. Splinter validates the output satisfies the objective before marking the step complete.

Eligibility & Signals

Only the eligible agent can act. Steps must declare when done. Idle agents must declare what they're waiting for. No silent stalls.

Checkpoints & Pipelines

State is checkpointed at every step. Resume from the last checkpoint instead of rerunning everything. Sequential order is enforced.
